千丁（Qianding）是龙湖集团旗下数字科技子公司。

## 核心目标

从目标公司的公开信息中提取“认知操作系统”（思维模型、决策启发式、价值观、风险盲区），生成可复用的公司视角 AI Skill，让千丁团队在战略分析和业务决策时复用优秀公司的思考方式。

## 当前工作流与能力边界

### 已支持的产品工作流

1. 读取本地已有的六路研究文件。
2. 汇总来源数、关键发现、数据缺口和认知提取进度。
3. 从各路 `*_cognitive.md` 文件中提取思维模型。
4. 应用三重验证（跨域/预测/排他），只将满足至少两项验证的模型写入最终 Skill。
5. 生成 `company-skill.md`，并可运行质量检查。
6. 提供 SEC、年报 PDF、财务基本面等辅助脚本，用于支持人工或半自动采集。
7. 将六路研究设计为“并行推进、逐路确认、统一汇总”的协作流程：在支持并行子任务的 AI 执行环境中，六路研究可以同时启动；每一路完成后必须单独向用户确认，全部确认后才能进入汇总和认知提取阶段。

### 真实能力边界

1. 信息可得性：如果公开资料、原始报告、管理层访谈或可靠第三方来源不足，不能把缺口包装成确定结论。
2. 来源质量：低质量、单一来源或相互矛盾的信息只能形成低置信度判断，需要在输出中明确标注。
3. 执行环境：并行六路研究依赖当前 AI 执行环境是否支持并行子任务；如果环境不支持，只能退化为串行执行、任务清单或处理已有文件。
4. 交付接口：Web Report、飞书、Obsidian 或线上站点写入尚未接入 `run_improved.py`，需要在生成 `company-skill.md` 后另行交付。
5. 本地 CLI 定位：`run_improved.py` 是研究文件处理入口，不是完整的研究代理调度器；它负责检查、汇总、验证和生成 Skill。

如果用户要求超出当前环境或接口支持的能力，必须先说明具体限制，再给出可执行的替代方案。

## 输入文件约定

默认研究目录为当前仓库下的 `output/`，也可以通过 `--research-dir` 指定。

建议文件命名：

```text
[company]_financials.md
[company]_business.md
[company]_strategy.md
[company]_stream_D.md
[company]_stream_E.md
[company]_stream_F.md

[company]_financials_cognitive.md
[company]_business_cognitive.md
[company]_strategy_cognitive.md
[company]_stream_D_cognitive.md
[company]_stream_E_cognitive.md
[company]_stream_F_cognitive.md
```

六路含义：

1. **A 财务**：营收、毛利率、亏损、现金流、盈利拐点。
2. **B 业务**：产品矩阵、技术壁垒、竞争格局、市场份额。
3. **C 战略**：演进路径、国际化、并购、上市策略。
4. **D 领导思维**：创始人/管理层决策逻辑、关键决策时间线。
5. **E 批判视角**：商业模式困境、竞争压力、运营风险。
6. **F 文化符号**：品牌叙事、社区文化、思想领导、组织暗规则。

## 执行流程

### 1. 身份确认

先确认公司唯一标识，不允许凭模糊简称直接开始：

```text
公司全称：
股票代码/上市地：
核心品牌/业务范围：
研究目录：
```

如果无法确认，停下来询问用户。不要自行猜测。

### 2. 并行采集或检查六路研究文件

如果用户已经提供研究文件，直接检查文件是否齐全。

如果文件缺失，明确说明缺失哪些流，并让用户选择：

```text
1. 继续处理已有文件
2. 先补齐缺失研究
3. 仅生成阶段性 Review
```

如果用户选择补齐缺失研究，默认采用“主管 + 六路研究员”的并行协作模式；这是产品流程设计，不是能力边界：

1. 主管一次性启动六路独立研究任务，不要按 A→B→C→D→E→F 串行执行。
2. 六路研究员分别负责：
   - A 财务：营收、毛利率、亏损、现金流、盈利拐点。
   - B 业务：产品矩阵、技术壁垒、竞争格局、市场份额。
   - C 战略：演进路径、国际化、并购、上市策略。
   - D 领导思维：创始人/管理层决策逻辑、关键决策时间线。
   - E 批判视角：商业模式困境、竞争压力、运营风险。
   - F 文化符号：品牌叙事、社区文化、思想领导、组织暗规则。
3. 每一路完成后，先向用户展示该路摘要、来源质量、关键不确定性和建议产物路径，并等待用户确认。
4. 用户可以对某一路提出返工要求；只重做该路，不影响其他已确认研究流。
5. 只有六路全部达到用户确认状态，才进入“汇总 Review”和“认知提取与三重验证”。
6. 如果当前环境不支持并行子任务，必须把限制说明为“执行环境限制”，再让用户选择：
   - 改为串行执行；
   - 只生成六路研究任务清单；
   - 先处理已有文件。

可用辅助脚本：

```bash
python scripts/fetch_sec_filings.py [company_or_ticker] --annual-reports
python scripts/edgar_xbrl_extractor.py [ticker] --years 3 --json --output output/[company]_financials.json
python scripts/extract_annual_report_data.py [annual_report.pdf] --mode all
python scripts/fundamentals_analyzer.py --input [metrics.json]
```

这些脚本只负责采集或结构化局部资料，不代表完整研究已完成。

### 3. 汇总 Review

使用：

```bash
python scripts/merge_research.py output --company [company]
```

检查内容：

1. 每路是否有原始研究文件。
2. 来源数量是否足够。
3. 是否有营收、利润、业务分部等关键数据缺口。
4. 是否有 `*_cognitive.md` 认知提取文件。
5. 是否存在未处理的矛盾观点。

如果总来源数过低、关键数据缺失或认知文件不足，先提示风险，不要直接包装成最终结论。

### 4. 认知提取与三重验证

使用：

```bash
python scripts/extract_mental_models.py output --company [company] --output output/[company]_skill.md
```

每条思维模型必须尽量包含：

```markdown
### [模型名称]
- **核心**：
- **来源证据**：
- **预测验证**：
- **排他性**：
- **confidence**: high/medium/low
```

三重验证规则：

1. **跨域验证**：同一模型在至少两个独立研究流中出现，或来源证据明确引用多个研究流。
2. **预测验证**：能解释至少一个已知历史决策，并说明依据。
3. **排他验证**：不是“客户至上/持续创新/追求卓越”这类通用表述，而是有公司特异性的认知模式。

只有满足至少两项验证的模型进入最终 Skill。未通过模型可以在工作记录中保留，但不要放入核心输出。

### 5. 质量检查

生成后运行：

```bash
python scripts/quality_check.py output/[company]_skill.md
```

至少检查：

1. 来源标注。
2. 千丁关联。
3. 思维模型数量。
4. 验证状态覆盖。
5. 不确定性/诚实边界。

如果检查失败，先修正输入材料或认知文件，再重新生成。

## 产出结构

最终 `company-skill.md` 应包含：

1. Mental Models（思维模型）
2. Decision Heuristics（决策启发式）
3. Values & Anti-patterns（价值观与禁忌）
4. Honest Limits（诚实边界）
5. Expression DNA（表达特征）
6. 千丁启示（认知迁移、风险对照、决策模式参考）
7. 证据链与验证总表
8. 使用限制

## 质量原则

1. 不把事实罗列伪装成思维模型。
2. 不把单源观点包装成高置信度结论。
3. 不强行调和矛盾观点。
4. 不输出“待人工补充”作为最终产物内容。
5. 对未知问题主动标注不确定性。
6. 所有关键判断必须能追溯到研究流或来源文件。
