# 三重验证规则

每条思维模型必须满足以下至少两项，方可进入最终 Skill：

## 1. 跨域出现（Cross-Domain）

**标准**：同一思维模型出现在 ≥2 个独立的研究流（Stream）中

**示例**：
- ✅ 通过：D 流（CEO 访谈） + C 流（决策文档）同时提到该模型
- ❌ 不通过：仅在 D 流出现，其他流无佐证

## 2. 预测能力（Predictive）

**标准**：能合理解释该公司 ≥1 个已知历史决策，吻合度 ≥70%

**测试**：
1. 选取 3 个已执行且公开解释的重大决策
2. 仅凭模型描述预测"为何做此决策"
3. 比对公开理由，计算吻合度
   - 80-100%：高度一致
   - 60-79%：部分一致
   - <60%：不一致
4. 平均吻合度 ≥70% → 通过

**失败处理**：任一案例 <50% → 返回补充调研

## 3. 排他性（Exclusivity）

**标准**：不是行业通用常识，而是该公司独特认知

**判断**：
1. 将表述去公司化（用"该公司"替换）
2. 问：能套用到行业前 3 名公司吗？
3. 能 → 标记 `[⚠️通用]`；不能 → 标记 `[✅排他]`

**示例**：
- "客户至上" → ❌ 通用
- "客户无理时也要服务" → ✅ 排他

## 验证状态速查

| 状态 | 是否进入核心 |
|------|-------------|
| `[✅跨域][✅预测][✅排他]` | ✅ 高权重 |
| `[✅跨域][✅预测][⚠️通用]` | ✅ 中权重 |
| `[✅跨域][⚠️无预测数据]` | ⚠️ 附录 |
| `[⚠️单源]` | ❌ 不进入 |
| `[❌预测失败]` | ❌ 丢弃 |
