# 思维模型提取指南

## 目标

从目标公司的公开信息中提取可操作的思维模型、决策启发式和价值观，用于生成可复用的 AI Skill。

## 提取维度

| 维度 | 关注点 | 信息来源 |
|------|--------|----------|
| 思维模型 | 高层决策框架（第一性原理、系统思维等） | 管理层演讲、访谈、内部信 |
| 决策启发式 | 可操作的经验法则（"当...时，总是..."） | 战略决策时间线、历史公告 |
| 价值观底线 | 明确不做的领域、厌恶的模式 | 批判视角分析、内部文化材料 |
| 表达特征 | 高频词汇、核心隐喻、叙事风格 | 公开讲话、全员信、投资者材料 |
| 风险盲区 | 批评者指控的弱点、未反驳的质疑 | 做空报告、媒体调查、客户投诉 |
| 组织记忆 | 反复提及的历史事件、组织神话 | 周年庆材料、内部故事、培训案例 |

## 提取原则

1.  **可操作性**：思维模型必须能指导决策（"遇到 X 时该怎么做"），而非描述性事实
2.  **可验证性**：每条模型必须标注来源和验证状态
3.  **独特性**：优先提取该公司独特诠释的认知，而非行业通用话术
4.  **矛盾保留**：不同来源的冲突观点全部记录，不强行统一

## 提取步骤

1.  **收集素材**：从 6 个研究流（财务/业务/战略/领导思维/批判/文化）收集原始信息
2.  **识别模式**：将事实陈述上升为思维框架（例："他们2023年收购X" → "当看到潜在竞争对手时，优先收购而非竞争"）
3.  **命名模型**：给模式起清晰、可复用的名称
4.  **标注证据**：记录具体来源（URL/文件名+位置）
5.  **应用验证**：通过三重验证（跨域/预测/排他）评估可信度

## 输出格式

```markdown
### [模型名称]
- **核心**：一句话描述该思维模型
- **来源证据**：[✅跨域] D流 CEO访谈 P12 + C流 2022年决策文档
- **预测验证**：[✅预测] 解释了某历史决策（吻合度 85%）
- **排他性**：[✅排他] 与行业通用做法不同
- **confidence**: high/medium/low
```

## 质量检查

- [ ] 每条模型都有明确来源
- [ ] 标注了验证状态（至少一项：跨域/预测/排他）
- [ ] 标注了置信度
- [ ] 矛盾观点有记录
- [ ] 模型具有可操作性（能回答"遇到某类问题时该怎么想"）
